# script-to-blog — 스크립트 파일 기반 블로그 포스팅

사용자가 직접 작성한 구조화된 스크립트 파일(`.md`)을 받아 블로그 포스트를 생성하고 발행 파이프라인까지 안내한다.

`/content-repurposer`와의 차이: source-analyst 단계 없음. 스크립트에 이미 글 구조·키워드·주의사항이 정의되어 있으므로 blog-writer가 그것을 재해석하지 않고 그대로 따른다.

---

## 트리거

- `/script-to-blog [파일경로]`
- "이 스크립트 파일로 블로그 포스팅해줘" + 파일경로
- "스크립트로 포스팅" + 파일경로

---

## 실행 모드

| 상황 | 모드 |
|------|------|
| 스크립트 파일에 편이 1개 | 단일 포스트 모드 |
| 스크립트 파일에 편이 2개+ | 시리즈 모드 (편별 순차 실행) |
| 특정 편만 지정 (`--part N`) | 해당 편만 실행 |

---

## Phase 1: 스크립트 파싱

1. 지정 파일을 읽는다
2. `# ✍️ N편 프롬프트:` 패턴으로 편을 분리한다 (없으면 전체를 1편으로 처리)
3. 각 편의 `[블로그 개요]` → 분량 지시 확인:
   - 예상 분량 > 3,500자 **또는** 코드블록 3개 이상 → 분리 권장 (사용자에게 알림)
   - 분리 지점: 스크립트의 `### 본론 N:` 경계 중 중간 지점
4. 이미 발행된 이전 편이 있으면 `_posts_index.md`에서 URL·thumb을 확인한다

---

## Phase 2: 편별 blog-writer 실행

각 편에 대해 아래 지시사항을 포함해 blog-writer 에이전트를 실행한다.

### 스크립트 모드 규칙 — blog-writer에 전달

```
[스크립트 모드 규칙]

1. 스크립트의 [글 구조] H2/H3 구조를 그대로 따른다 — 임의 재구성 금지
2. 스크립트의 [작성 시 주의사항] 모든 지시사항 준수
3. 스크립트 지정 SEO 키워드를 제목·소제목에 자연스럽게 포함
4. 스크립트에 제공된 코드블록은 그대로 사용, 임의 수정 금지
5. [대괄호] 표시 부분은 스크립트 문맥에 맞게 구체화
6. 시리즈 전편 예고·다음 편 예고는 스크립트 마무리 섹션 지시를 따름

[금융 콘텐츠 규칙 비적용]
기술/AI 콘텐츠이므로 아래 항목은 적용하지 않는다:
- 최상단 날짜·주의 문구 불필요
- 공식 출처 표 불필요
- 금융 기관 URL 불필요

[탈AI 규칙 — 금융 포스트와 동일 수준으로 전부 적용]
- 서론 첫 문단: 구어체 도입 필수 ("~거든요", "~더라고요", "~했는데")
- 본문 중간: 개인 경험·주변 사례 문단 1회 이상 포함
- 결론 마지막 문장: 작성자 의견 포함 ("개인적으로는 ~", "솔직히 ~")
- AI 정형화 표현("이러한", "따라서", "이에 따라", "살펴보겠습니다") 2회 미만
- "함께 알아보겠습니다", "아래에서 자세히", "이번 포스팅에서는", "마지막으로" 0회
- 단락 4줄 이상 연속 없음 / H2 내 H3 없는 단락 5개 이상 없음
- 임팩트 단문 단락(독립 1~2문장) 1개 이상 / 괄호 감정 주석 1~2개

[메타 블록쿼트 — 파일 최상단 H1 바로 아래]
> **카테고리**: [스크립트 지정 카테고리, 없으면 소프트웨어 비교]
> **태그**: [스크립트 지정 SEO 키워드 기반 10개 내외]
> **메타 디스크립션**: [155자 이내]
> **읽기 시간**: [분]분

[THUMBNAIL_PROMPT] 블록 파일 맨 끝에 포함:
[THUMBNAIL_PROMPT]
[영어, 16:9 비율, no text, 포스트 주제와 관련된 시각적 이미지]
[/THUMBNAIL_PROMPT]
```

### 내부 링크 — 함께 읽으면 좋은 글

`_posts_index.md`에서 **같은 카테고리 + 관련 카테고리** 포스트를 최대 3개 선택한다.

- 기술/AI 콘텐츠의 경우: `소프트웨어 비교` + `AI 도구활용` 카테고리 포스트 포함
- 시리즈물의 경우: 이전 편을 반드시 1번 카드로 포함 (발행 완료 후 URL 확보 전제)
- `thumb: pending` 포스트는 제외

---

## Phase 3: quality-reviewer 검증

blog-writer 결과물에 대해 quality-reviewer를 실행한다.

기술 콘텐츠 전용 추가 체크:
- [ ] 코드블록이 스크립트 원본과 일치하는가
- [ ] [대괄호] 플레이스홀더가 모두 채워졌는가
- [ ] 시리즈 연결 문구(이전 편 언급, 다음 편 예고)가 자연스러운가
- [ ] 탈AI 규칙이 금융 포스트와 동일 수준으로 적용됐는가

---

## Phase 4: 파이프라인 안내

각 편 작성 완료 후 출력:

```
✅ 발행 준비 완료
제목: [H1 제목]
파이프라인:
  1. python -X utf8 04_notion_upload.py
  2. python -X utf8 03_tistory_playwright.py
  3. OG 이미지 수집 + _posts_index.md 업데이트
```

> ⚠️ 시리즈인 경우: 다음 편 작성 전에 반드시 이 편의 OG 이미지를 수집하고
> `_posts_index.md`를 업데이트해야 한다. 이전 편 thumb이 `pending`이면
> 다음 편의 내부 링크 카드가 깨진다.

## 에러 처리

| 상황 | 처리 |
|------|------|
| 스크립트 파일 없음 | 경로 오류 안내 후 중단 |
| 구분자 패턴 없음 | 전체를 1편으로 처리 |
| 분량 초과 예상 | 분리 지점 제안 후 사용자 확인 |
| 이전 편 thumb pending | ⚠️ 경고 출력 후 내부 링크 카드 제외하고 진행 |
