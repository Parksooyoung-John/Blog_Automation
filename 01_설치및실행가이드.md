# 설치 및 실행 가이드

## ⚠️ 티스토리 Open API 종료 안내
티스토리 Open API는 **2024년 2월 완전 종료**되었습니다.
대신 **Playwright 브라우저 자동화** 방식을 사용합니다.

---

## 1. Python 패키지 설치

```bash
pip install playwright requests python-dotenv --break-system-packages
playwright install chromium
```

---

## 2. 환경변수 설정

`.env.example` 파일을 복사해 `.env`로 저장 후 실제 값 입력:

```bash
cp .env.example .env
```

입력 항목:
- `TISTORY_KAKAO_EMAIL` : 티스토리 카카오 로그인 이메일
- `TISTORY_KAKAO_PASSWORD` : 카카오 비밀번호
- `TISTORY_BLOG_NAME` : 블로그 주소의 서브도메인 (xxx.tistory.com → xxx)
- `NOTION_API_KEY` : Notion Integration Token
- `NOTION_DATABASE_ID` : Notion DB ID (URL 마지막 32자리)
- `OPENAI_API_KEY` : OpenAI 키 (썸네일 자동생성, 없으면 생략)

---

## 3. Notion DB 컬럼 구성

스크립트 저장 DB에 아래 컬럼이 필요합니다:

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| 제목 | Title | 포스팅 제목 |
| 상태 | Select | `발행대기` / `처리중` / `발행완료` / `오류` |
| 태그 | Multi-select | 태그 목록 |
| 카테고리 | Select | 티스토리 카테고리명 |
| 발행URL | URL | 발행 후 자동 기입 |
| 처리완료시각 | Date | 발행 후 자동 기입 |

---

## 4. Make.com 연동 (Notion 상태 자동 변경)

Make.com 시나리오 마지막 단계에서:
- Notion 페이지 `상태` 컬럼을 **`발행대기`** 로 업데이트

→ Python 스크립트가 이를 감지해 자동 발행

---

## 5. 실행

```bash
python 03_tistory_playwright.py
```

처음 실행 시 브라우저 창이 열리며 동작 확인 가능.  
안정화 후 `headless=True`로 변경하면 백그라운드 실행.

---

## 6. 자동 스케줄 실행 (Windows 작업 스케줄러)

**방법 A: Windows 작업 스케줄러**
1. `작업 스케줄러` 열기
2. `작업 만들기` → 트리거: 매일 오전 9시
3. 동작: `python C:\경로\03_tistory_playwright.py`

**방법 B: Make.com Webhook 트리거**
- Make.com에서 글 생성 완료 시 → 로컬 PC의 webhook 호출
- 로컬에서 `flask` 등으로 간단한 수신 서버 운영

---

## 전체 파이프라인 요약

```
Make.com (스크립트 생성)
    └→ Notion DB 상태: "발행대기"

Python Playwright (03_tistory_playwright.py)
    └→ Notion에서 "발행대기" 감지
    └→ DALL-E 썸네일 생성
    └→ 쿠팡 링크 삽입
    └→ 티스토리 브라우저 자동 발행
    └→ Notion 상태: "발행완료" + 발행URL 기록
```
